
Derived queries are methods in a Spring Data repository interface where the method name itself defines the query.

Spring Data JPA analyzes the method name and automatically creates the corresponding JPQL / SQL query behind the scenes — no need to write it manually.

**Example:**

```
// Find by exact match
List<User> findByUsername(String username);

// Find by multiple fields
User findByUsernameAndEmail(String username, String email);

// Find users older than 25
List<User> findByAgeGreaterThan(int age);
```

Spring converts these into SQL-like queries automatically:

```
SELECT * FROM user WHERE username = ?;
SELECT * FROM user WHERE username = ? AND email = ?;
SELECT * FROM user WHERE age > ?;
```

---
### Common Derived Query Keywords

Here’s a cheat sheet of keywords you can use in derived method names:

| Keyword                              | Description         | Example                                         |
| ------------------------------------ | ------------------- | ----------------------------------------------- |
| And                                  | Combines conditions | findByUsernameAndEmail                          |
| Or                                   | Either condition    | findByUsernameOrEmail                           |
| Is, Equals                           | Equality            | findByEmail or findByEmailEquals                |
| Between                              | Range               | findByAgeBetween(int start, int end)            |
| LessThan, GreaterThan                | Comparison          | findByAgeGreaterThan(int age)                   |
| Like, NotLike                        | Pattern matching    | findByUsernameLike(String pattern)              |
| StartingWith, EndingWith, Containing | String matching     | findByEmailContaining("@gmail.com")             |
| In, NotIn                            | Set membership      | findByIdIn(List<Long> ids)                      |
| OrderBy                              | Sorting             | findByAgeGreaterThanOrderByUsernameAsc(int age) |
| True, False                          | Boolean             | findByActiveTrue()                              |
| IsNull, IsNotNull                    | Null checks         | findByEmailIsNull()                             |

---
### Limitations

| Limitation                           | Explanation                                                                                  |
| ------------------------------------ | -------------------------------------------------------------------------------------------- |
| Complex queries become unreadable    | findByUsernameAndEmailAndAgeGreaterThanAndStatusTrue → messy.                                |
| No joins beyond entity relationships | <span style="color:red;">For multi-table joins or subqueries, use JPQL or native SQL.</span> |
| Limited dynamic conditions           | You can’t add runtime conditions easily (use Specification or QueryDSL for that).            |
| Runtime validation only              | Method names are parsed at runtime — typos cause startup errors                              |
